HelloThere

18th November 2017 at 1:13pm

This is an HTML+Wikitext Table Renderer plugin for TiddlyWiki (http://tiddlywiki.com) by Joshua Fontany.

It contains a macro <<table>> which calls a RenderTable Widget that renders an HTML Table based on a "schema tiddler" parameter. The schema tiddler is a JSON tiddler and should follow a specific format. See Table Schemas for more information.

See Examples/Macros/Table for examples of calling the plugin with different schemas and datafilters, and using the plugin's Sort and Filter functions.

See Examples/Table-Edit for an advanced example using the Render Table widget to render parts of Tiddlywiki's User Interface.

    !!! Rendered via Tiddlywiki's View and Edit Template functions. !!

    Edit this tiddler to see the examples function. While editing, preview the text field (closed/open eye button) to switch a table with this setup to "edit" mode while it is in the text field being edited:

    data-_0_: testing
    data-_1_: test
    data-_2_: THIS IS INTERESTING
    data-_3_: 555555
    data-_4_: 123123123

    This tiddler is tagged: EditTableTest

    This causes a template defines in Examples/Template to be inserted into the tiddler's layout above the text field. The positioning is set in the list field of the $:/tags/ViewTemplate and $:/tags/ViewTemplate tiddlers.

    This is the macrocall that renders the table, found in Examples/Template:

    <$macrocall $name="table" schema="Examples/Table-Edit_schema.json" mode="array" datafilter="[[Examples/Table-Edit_data.json]]"/>

    The macro renders a table based on the Examples/Table-Edit_schema.json schema tiddler, which defines a template for each column. The data for the table, found in Examples/Table-Edit_data.json, defines an array of field names. Each template uses this field name to reference a field on the <<currentTiddler>>. The <<CellTemplate>> macro for the second column, found in Examples/Template/Table-Edit/value, changes the rendering behavior when the <<currentTiddler>> has the draft.of field (i.e. a tiddler is being displayed in edit mode).

    Table Schemas

    18th November 2017 at 1:21pm

    Table Schemas are a model of a "HTML +Wikitext Table", which are defined in JSON Tiddlers according to a specific pattern of Key:Value pairs.

    The Render Table Widget (usually called by the <<table>> macro) accepts a "schema" attribute, and uses the provided schema to render the table. Schemas can define CSS classes and styles for various elements, hide or reveal certain parts of the table, add fully customizable headers and footers, and set template tiddlers to render table cells through.

    A Schema must be a "flat" (one layer deep) Json object with the following properties:
    • An array of fields[\n+].@name properties.
    • A table.@primarykey property, which defines the primary key (unique) of the table.

    All properties of the fields array besides @name are optional, and begin with the "root path" for each field fields[\n+]. plus the field name:

    • caption - Optional text to display instead of the field's @name in the automatically generated header or footer cells.
    • description - Displayed in a popup shown by clicking on the field's caption (header or footer).
    • field-class - An optional CSS class to assign to cells in this column.
    • field-style - An optional CSS style to assign to cells in this column.
    • footer-class - An optional CSS class to assign to the footer cell in this column.
    • footer-style - An optional CSS style to assign to the footer cell in this column.
    • header-class - An optional CSS class to assign to the header cell in this column.
    • header-style - An optional CSS style to assign to the header cell in this column.
    • template - The name of a CellTemplate Tiddler to use to render the data cells in this column.

    All properties of the table object besides @primarykey are optional. All properties begin with the "root path" for the table objecttable. plus the field name:

    • caption - Text displayed above the table.
    • caption-class - An optional CSS class to assign to the caption.
    • caption-style - An optional CSS style to assign to the caption.
    • class - An optional CSS class to assign to the table element.
    • description - Displayed in a popup shown by clicking on the table's caption.
    • filterable - Set to true to mark the table as filterable.
    • footers - Blank = false (default). Set to true to render an auto-footer for each column.
    • footers-class - An optional CSS class to assign to the footer cells.
    • footers-style - An optional CSS style to assign to the footer cells.
    • headers - Blank = true (default). Set to false to remove the auto-header for each column.
    • headers-class - An optional CSS class to assign to the header cells.
    • headers-style - An optional CSS style to assign to the header cells.
    • sortable - Set to true to mark the table as sortable.
    • style - An optional CSS style to assign to the table element.

    The table schema may also have a set of properties defining optional headers or footers. These are rendered before the automatically generated headers or after the automatically generated footers. Optional headers or footers are defined as nested array properties that define rows and columns, header[\n+].col[\n+].. Wach cell of the array can have 3 properties, colspan which defines the colspan property of the header tag, rowspan which does the same for the rowspan property, and template which is the name of the template tiddler transcluded into that header or footer cell.

    For example, a header cell that spans 4 columns and renders the Examples/HeaderTest tiddler is defined below. The rowspan property was left out as it is not needed. 

    {
...
"headers[0].col[0].colspan": "4",
"headers[0].col[0].template": "Examples/HeaderTest",
...
}
